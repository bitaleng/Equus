‘설정항목 추가’ 이미지 참조

- '지출관리' 페이지 바로가기 추가
- '정산하기' 페이지 바로가기 추가

1. ‘지출관리’페이지

1) 지출입력창 : 지출항목, 날짜, 시간, 금액을 입력할 수 있는 입력창을 만들어 줘
여기서 입력한 값으로 누적된 합을 정산시간기준으로 매출집계에서도 보여줘

‘매출 집계’ 박스 하단 보강

a. 지출항목추가  지출금액 : 총      원
b. 오른쪽이나 아래에 지출입력창을 불러오는 작은 버튼 만들기
[지출 입력](작은 플로팅 다이얼로그: 항목과 금액 입력 가능하도록 만들어야함)
지출관리 페이지에 있는 지출입력과 동일한 작업
여기서 입력한 내용이 관리자설정의 지출섹션에 기록되게 함
[정산 페이지로](정산 탭 이동, 관리자 비밀번호 입력)

2) 지출목록
지출입력창에서 입력한 정보를 입력창 아래에 데이타 베이스처럼 테이블로 만들어줘(상세기록이랑 비슷하게)
-열제목 : 지출항목 / 날짜 / 시간 / 금액 / 수량/ 
여기서 입력된 지출금액이 정산페이지에서 사용될거야,


2. [정산하기] 페이지 만들기

참고용어
- 시작 시재금(Opening Float) :오늘 장사 시작할 때 금고에 넣은 잔돈
- 기대현금(Expected Cash) :오늘 장사 끝났을 때 계산상 금고에 있어야 할 돈	
- 목표 시재금(Target Float) :내일 아침 장사 시작할 때 금고에 남길 돈
- 은행 입금액 :	기대현금 – 목표시재금

화면 레이아웃은 기존 ‘오늘 현황’ 패널 구조와 비슷하게 세로 3단 카드로 구성.


1) 상단카드-정산기본정보
- 정산기간 전일 10:00 ~ 오늘 10:00 (자동 표시)(날짜와 시간)
- 시작 시재금 : "전일 마감에 확정된 목표시재금"으로 자동채움
  시작시재금은 수정가능하게도 만들어줘(수정시 변경이력남김)
- 목표 시재금 입력 : 금일 장사에 준비할 시재금(수정가능)

2) 중상단카드-자동집계결과 요약
이 부분은 매출집계와 같은 형태로 만들어 줘
(카드 레이아웃 또는 테이블 형식)

[입실요금합계]  
① 일반요금합계 : 현금___ / 카드___  / 이체___
② 추가요금합계 : 현금___ / 카드___  / 이체___

3) 하단카드: 실잔액 및 과부족카드

- ⑪금전함실잔액 : 직원이 금전함에서 현금을 전부 세어서 계산된 금액을 직접입력
- ⑫기대잔액 = ⑬(현금총매출) - ⑭(현금총지출) + 시재금
- 과부족 = 실잔액-기대잔액(0이 아닌경우 현금흐름에 빠지거나 더해진게 있는 경우임)
- ⑮매출에서 목표시재금에 포함시킨 금액 = 목표시재금 - 시재금

- 버튼1 : 은행입금금액 산출(실잔액(=기대잔액)-목표시재금(시재금+⑮매출 일부))
- 버튼2 : '정산확정;
closing_day 레코드 저장
다음 기간 opening_float = 목표 시재금 자동 설정
과부족 0이 아닐 경우 메모 없이는 확정 불가
결과 확정 후에는 읽기전용 요약 화면 표시 → “정산완료” 상태로 변경.
3. 기타 수정
- 기본 UI 화면에서 오전 10시 정산 시점에 자동 으로 팝업으로 “정산 시간입니다” 알림창 띄우기('닫기' 누르기전에 계속 띄워놓기)
